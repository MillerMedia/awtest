### Exact Release Workflow to Implement

```yaml
# .github/workflows/release.yml
name: Release
on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: actions/setup-go@v5
        with:
          go-version: '1.19'

      - name: Run tests
        run: go test ./...

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          version: '~> v2'
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### Critical Implementation Details

**Use latest stable action versions:**
- `actions/checkout@v4` (not v3 — v3 uses Node 16 which is deprecated)
- `actions/setup-go@v5` (not v4 — v5 is latest stable)
- `goreleaser/goreleaser-action@v6` (supports GoReleaser v2.x)

**`fetch-depth: 0` is required** for the release workflow. GoReleaser needs full git history to generate changelogs from tags. Without it, GoReleaser will fail or produce empty release notes.

**`permissions: contents: write`** is required for the release workflow so the `GITHUB_TOKEN` can create releases and upload assets.

**GoReleaser `--clean` flag:** Replaces the deprecated `--rm-dist` flag in GoReleaser v2.x.

**Test workflow uses `go test` and `go build` directly** rather than `make test`/`make build` because:
- CI doesn't need coverage reports or race detector overhead for every push
- Avoids dependency on Make being available (though ubuntu-latest has it)
- Keeps CI simple and fast

**GoReleaser checksums:** SHA256 checksums file is generated automatically by GoReleaser. The default produces a `awtest_<version>_checksums.txt` file. No additional `checksum` config needed in `.goreleaser.yaml`.

**Brews section already configured** in `.goreleaser.yaml` — GoReleaser will automatically update the `MillerMedia/homebrew-tap` repo on release. This requires that the `GITHUB_TOKEN` has permissions to push to that repo (it will if both repos are under the same owner).
